ğŸ“– What is Embrix O2X?

Embrix O2X (Order-to-X) is a comprehensive billing and order management platform designed for telecommunications operators, SaaS companies, and multi-tenant service providers. It handles the complete customer lifecycle from order entry through provisioning, usage tracking, billing, invoicing, payment processing, and revenue recognition.

Key Capabilities

  • âœ… Multi-Tenant Architecture - Support multiple customers/brands on shared infrastructure
  • âœ… Order Management - Process complex orders with workflow automation
  • âœ… Service Provisioning - Automate service activation on network elements
  • âœ… Usage Processing - Handle millions of CDRs with real-time and batch rating
  • âœ… Flexible Billing - Recurring, usage-based, tiered, bundled pricing models
  • âœ… Revenue Recognition - IFRS 15 and ASC 606 compliant
  • âœ… Payment Processing - Multiple payment gateways (Braintree, Stripe, PayPal)
  • âœ… Tax Compliance - Avalara integration and CFDI (Mexico) support

Infrastructure Setup

Start required infrastructure services: 
# PostgreSQL
docker run -d --name postgres \
  -p 5432:5432 \
  -e POSTGRES_USER=omsadmin \
  -e POSTGRES_PASSWORD=omsadmin \
  -e POSTGRES_DB=omsdevdb \
  postgres:10.5

# Redis
docker run -d --name redis -p 6379:6379 redis:latest

# ActiveMQ
docker run -d --name activemq \
  -p 61616:61616 \
  -p 8161:8161 \
  webcenter/activemq:5.14.3

# HashiCorp Vault (dev mode)
docker run -d --name vault \
  -p 8200:8200 \
  -e VAULT_DEV_ROOT_TOKEN_ID=myroot \
  vault:latest

Build Order

Due to module dependencies, build in this order: 
# 1. Foundation
cd common && mvn clean install
cd ../engine && mvn clean install
cd ../gateway-common && mvn clean install
cd ../oms-component && mvn clean install
cd ../jobs-common && mvn clean install

# 2. Core Services
cd ../core/service-billing && mvn clean install
cd ../service-invoice && mvn clean install
cd ../service-usage && mvn clean install
cd ../service-payment && mvn clean install
cd ../service-revenue && mvn clean install
cd ../service-mediation && mvn clean install
cd ../service-transactional && mvn clean install
cd ../service-sso && mvn clean install
cd ../service-proxy && mvn clean install
cd ../batch-process && mvn clean install

# 3. Gateways
cd ../../crm_gateway && mvn clean install
cd ../provision_gateway && mvn clean install
cd ../tax-gateway && mvn clean install
cd ../payment-gateway && mvn clean install
cd ../finance-gateway && mvn clean install

Running Services

# crm_gateway (main API)
cd crm_gateway
mvn spring-boot:run -Dspring.profiles.active=dev

# Access GraphQL Playground
open http://localhost:8080/graphiql

# ActiveMQ Console
open http://localhost:8161 (admin/admin)

# Vault UI
open http://localhost:8200/ui (token: myroot)

Frontend Development

# ui-core (main admin app)
cd ui-core
npm install
npm start  # http://localhost:3000

# selfcare (customer portal)
cd selfcare
npm install
npm start  # http://localhost:3001

GraphQL API

# Create Account
mutation {
  processNewAccount(
    account: {
      accountName: "TechCorp Inc"
      accountType: BUSINESS
      billingAddress: { ... }
    }
  ) {
    accountId
    accountName
  }
}

# Create Order
mutation {
  createOrder(
    order: {
      accountId: "ACC-001"
      orderType: NEW
      serviceLines: [ ... ]
    }
  ) {
    orderId
    status
  }
}

# Search Usage
query {
  searchUsageRecords(
    accountId: "ACC-001"
