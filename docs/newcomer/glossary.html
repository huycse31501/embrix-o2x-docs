Platform Terms

Embrix O2X

Definition: Order-to-X platform - an enterprise billing and order management system for telecommunications and service providers.

Multi-Tenant

Definition: Architecture model where a single instance of the application serves multiple customers (tenants), each with isolated data and configurations.

Tenant

Definition: A customer organization using the Embrix O2X platform. Each tenant has isolated data, configuration, and branding.

Hub

Definition: A domain-specific module in the engine that encapsulates business logic for a particular area (e.g., billingHub, customerHub).

Engine

Definition: The core shared library containing all business logic organized into hubs. Used by all services.

Common

Definition: Shared library containing DTOs, enums, and constants used across all modules.

Billing and Finance

Account

Definition: A customer entity that receives services and bills. Can be Business (B2B) or Residential (B2C).

Subscription

Definition: A customer's active service instance with recurring charges, linked to a product and price offer.

Order

Definition: A request to add, modify, or cancel services for an account. Types include NEW, CHANGE, CANCEL, SUSPEND, RESUME.

Invoice

Definition: A bill sent to a customer detailing charges, taxes, and payment due. Statuses: DRAFT, OPEN, PARTIALLY_PAID, PAID, CANCELLED.

Charge

Definition: A line item representing a fee for services. Types: RECURRING, ONE_TIME, USAGE_BASED, ADJUSTMENT.

Payment

Definition: Money received from a customer, allocated to invoices to reduce balance.

Payment Allocation

Definition: The process of applying a payment amount to one or more invoices.

Credit Balance

Definition: Overpayment or credit amount that can be applied to future invoices.

Proration

Definition: Calculating partial charges when services start/end mid-billing cycle.

Bill Cycle

Definition: The recurring period for generating invoices (e.g., monthly, quarterly). Identified by bill_cycle_day (1-28).

Bill Unit

Definition: A billing entity that groups subscriptions for a single invoice.

Price Offer

Definition: The pricing configuration for a product, including recurring charges, one-time fees, and usage-based rates.

Bundle

Definition: A package of multiple products sold together, often at a discounted price.

Discount

Definition: Price reduction applied to charges. Types: PERCENTAGE, FIXED_AMOUNT, TIERED.

Technical Terms

JOOQ

Definition: Java Object Oriented Querying - a type-safe database access library that generates Java code from database schemas.

Flyway

Definition: Database migration tool that version-controls schema changes through SQL scripts.

GraphQL

Definition: Query language for APIs allowing clients to request exactly the data they need.

ActiveMQ

Definition: Java message broker implementing JMS for asynchronous communication between services.

Apache Camel

Definition: Integration framework for routing and transformation of messages between systems.

Vault (HashiCorp)

Definition: Secrets management tool for storing sensitive configuration (passwords, API keys, certificates).

Redis

Definition: In-memory data store used for caching and session management.

OAuth2

Definition: Authorization framework allowing third-party applications to obtain limited access to services.

JWT (JSON Web Token)

Definition: Compact token format for securely transmitting information between parties as JSON.

Groovy

Definition: Dynamic JVM language with Java compatibility, used for business logic in Embrix O2X.

DSL (Domain Specific Language)

Definition: In JOOQ context, the fluent API for constructing type-safe SQL queries.

Architecture Patterns

Event-Driven Architecture

Definition: Design pattern where components communicate through events (messages) rather than direct calls.

Gateway Pattern

Definition: An intermediary service that provides a single entry point for external system integration.

Shared Engine Pattern

Definition: Embrix O2X's approach of centralizing business logic in a shared library (engine) used by all services.

Hub-Based Organization

Definition: Organizing business logic into domain-specific modules (hubs) within the engine.

OMS (Order Management System)

Definition: Component library providing order orchestration framework using Apache Camel.

Message Queue

Definition: Asynchronous communication mechanism where messages are stored until processed by consumers.

Dead Letter Queue (DLQ)

Definition: Queue for messages that failed processing after maximum retry attempts.

Business Domains

AR (Accounts Receivable)

Definition: Managing customer balances, payments, collections, and aging.

Billing

Definition: Process of calculating charges and generating invoices for customer accounts.

Mediation

Definition: Collecting, normalizing, and validating raw usage records (CDRs) from network elements.

Rating

Definition: Applying pricing rules to usage records to calculate charges.

Usage

Definition: Customer consumption of services (voice calls, data, SMS, API calls).

CDR (Call Detail Record)

Definition: Record of a telecommunications transaction (call, SMS, data session) including duration, destination, timestamps.

Provisioning

Definition: Activating services on network elements or equipment (e.g., activating fiber ONT, configuring SIP trunk).

Revenue Recognition

Definition: Accounting process of recording revenue according to standards (IFRS 15, ASC 606).

Deferred Revenue

Definition: Revenue collected but not yet earned (e.g., prepaid annual subscription).

Collections

Definition: Process of pursuing payment from customers with overdue balances.

Dunning

Definition: Systematic process of communicating with customers to collect overdue payments.

B2C

Database Terms

Schema

Definition: Logical grouping of database objects (tables, views, functions). Embrix O2X uses multiple schemas: core_engine, core_billing, core_usage, etc.

Partitioning

Definition: Splitting large tables into smaller physical pieces for performance. Usage_record table is partitioned by date.

Index

Definition: Database structure improving query performance by providing fast access paths to data.

Foreign Key

Definition: Column referencing primary key of another table, enforcing referential integrity.

Migration

Definition: Version-controlled database schema change, managed by Flyway.

Transaction

Definition: Unit of work that must complete entirely or not at all (ACID properties).

Integration Terms

Webhook

Definition: HTTP callback triggered by events, allowing real-time notifications to external systems.

SFTP

Definition: Secure File Transfer Protocol - Encrypted protocol for file transfer.

Diameter Protocol

Definition: AAA (Authentication, Authorization, Accounting) protocol for real-time charging in telecom networks.

SOAP

Definition: Simple Object Access Protocol - XML-based messaging protocol for web services.

WSDL

Definition: Web Services Description Language - XML description of web service interfaces.

Service Names

service-billing

Definition: Microservice handling billing cycle execution and charge calculation.

service-invoice

Definition: Microservice generating invoice documents (PDF, HTML, XML).

service-usage

Definition: Microservice storing and querying usage data.

service-mediation

Definition: Microservice ingesting and normalizing raw CDRs.

service-payment

Definition: Microservice processing payments and managing payment methods.

service-revenue

Definition: Microservice handling revenue recognition and GL postings.

service-transactional

Definition: Microservice providing real-time GraphQL API for operations.

service-sso

Definition: Microservice handling authentication and authorization.

service-proxy

Definition: API gateway routing requests to appropriate services.

batch-process

Definition: Service running scheduled batch jobs (billing, usage processing, reports).

crm_gateway

Definition: Gateway integrating with CRM systems (Salesforce, MS Dynamics), main GraphQL API entry point.

provision_gateway

Definition: Gateway for service provisioning on network elements (Nokia, Cisco, ServiceNow).

tax-gateway

Definition: Gateway integrating with tax providers (Avalara, PAC for CFDI).

payment-gateway

Definition: Gateway integrating with payment processors (Braintree, Stripe, PayPal).

finance-gateway

Definition: Gateway integrating with ERP systems (QuickBooks, NetSuite, SAP).

Customer Journey Terms

Onboarding

Definition: Process of setting up a new customer account, selecting services, and activating them.

Activation

Definition: Moment when ordered services become active and usable by customer.

Suspension

Definition: Temporary deactivation of services, typically due to non-payment.

Churn

Definition: Customer cancelling service and leaving the platform.

Upsell

Definition: Selling higher-tier or additional services to existing customer.

Downsell

Definition: Customer moving to lower-tier service.

Performance Terms

Throughput

Definition: Number of transactions or operations processed per unit of time.

Latency

Definition: Time delay between request and response.

Concurrency

Definition: Number of simultaneous users or operations system can handle.

Connection Pool

Definition: Cache of database connections maintained for reuse to improve performance.

Cache Hit

Definition: When requested data is found in cache, avoiding database query.

Cache Miss

Definition: When requested data is not in cache, requiring database query.

Quality Assurance Terms

Unit Test

Definition: Test of individual code component in isolation.

Integration Test

Definition: Test of interaction between multiple components or systems.

E2E Test

Definition: Test simulating complete user journey through entire system.

Regression Test

Definition: Re-running tests to ensure changes haven't broken existing functionality.

Test Coverage

Definition: Percentage of code exercised by tests.

Mock

Definition: Simulated object replacing real dependency in tests.

Stub

Definition: Simplified implementation of interface for testing purposes.

Deployment Terms

Docker

Definition: Platform for developing, shipping, and running applications in containers.

Kubernetes

Definition: Container orchestration platform for automating deployment, scaling, and management.

Helm

Definition: Package manager for Kubernetes, used for deploying Embrix O2X services.

Pod

Definition: Smallest deployable unit in Kubernetes, containing one or more containers.

Namespace

Definition: Virtual cluster within Kubernetes for isolating resources.

ConfigMap

Definition: Kubernetes object storing configuration data as key-value pairs.

Secret

Definition: Kubernetes object storing sensitive data (passwords, tokens).

Ingress

Definition: Kubernetes resource managing external access to services.
